I. Pruning

Pruning Decides When A Tree Should Stop Growing

Pruning is the answer to three practical questions: how big should the tree be, why should we stop early, and when should we choose a smaller tree over a perfect fit to the training data.

What It Is
Cα(T) = Error(T) + α |T|

Cost-complexity pruning trades off fit and size. Smaller trees are easier to explain and less likely to chase random wiggles.

Why Do It

A tree with too many leaves can memorize the training data. Pruning keeps the model honest, simpler, and more stable on new data.

When To Choose It

Use the minimum cross-validated error, or the 1-SE rule when you want a smaller tree that performs almost as well as the best one.

R And Python Knobs

In R, cp and the cross-validation table guide pruning. In Python, the same idea appears through max_depth, min_samples_leaf, min_samples_split, and ccp_alpha.

What You Should See

A smaller tree with fewer branches, a shallower decision path, and test-set metrics that are often better than the unpruned version.

I. Split Mechanics

Classification Trees Choose Splits That Reduce Impurity Fastest

The first squirrel split works because the child nodes are more homogeneous than the parent, and the impurity formulas make that gain measurable.

Gini
Gini = 1 - Σ pk2

For a two-class problem, low Gini means the node is mostly one class. A tree searches for the split that gives the biggest drop in impurity.

Entropy
Entropy = - Σ pk log pk

Entropy tells the same story with a different scale: more mixed classes create more uncertainty, so good splits reduce that uncertainty.

Worked Example
1/2 - 1/6 = 1/3

In a simple two-group example, weighted Gini can drop from 0.5 to 0.1667, which is a large impurity reduction. That is the same logic behind a split such as Feeding in the squirrel tree.

How To Read A Tree

Each observation starts at the root, follows one branch at every rule, and ends in one leaf. One path equals one prediction.

Why This Matters Here

Squirrels make the logic visible because “ground or above ground” is intuitive enough to connect the formula directly to the branches.

II. Regression Trees

Regression trees

Regression trees divide predictors into non-overlapping regions, then predict the average response inside each region.

How To Set Up
  • Divide predictors into non-overlapping regions.
  • Use X1, X2, X3, and other predictors to split the space.
  • Choose splits by minimizing RSS, the residual sum of squares.
Split Objective
R1(j, s) = {X | Xj < s}
R2(j, s) = {X | Xj ≥ s}

The goal is to minimize the RSS after the split by finding the predictor j and threshold s that create the best regions.

Σi:Xi∈R1(j,s) (yi - ŷR1)2 + Σi:Xi∈R2(j,s) (yi - ŷR2)2
Prediction Rule

For every observation in region Rj, predict the mean response of the training observations in that region.

Advantages

Single trees are easy to interpret and expose the split logic directly.

Disadvantages

They can have low accuracy, and small data changes can lead to very different trees.

Why It Matters

Regression trees are a clear baseline: they show how the data is partitioned, but they also make the need for stronger ensembles easy to see.

II. Regression Trees

Regression Trees Replace Impurity With RSS And Predict Regional Means

The same math explains both the squirrel perch-height model and the wildfire acreage baseline: split the space into regions, then predict the mean within each region. The units stay visible in the error metrics, so you can see what the model is missing.

Split Objective
RSS = Σ (yi - ŷRj)2

Regression trees partition the predictor space into regions Rj and predict the mean response inside each region.

Metrics Primer
MAE and RMSE

MAE keeps the original unit, while RMSE punishes larger misses more heavily. Lower is better for both.

II. Why These Terms Matter

Why We Use Classification, Regression, Rules, And Benchmarking

This page explains the four ideas that organize the squirrel section. Each one answers a different modeling question, and together they show why a single dataset can teach a whole tree family.

Classification

Classification predicts a category such as Ground Plane or Above Ground. We use it when the answer is a label, not a number.

Regression

Regression predicts a numeric response such as perch height. We use it when the answer has units and the error should be measured in those same units.

Rules

Rules rewrite the tree as a readable if-then list. We use them when the audience needs the logic in compact language rather than a branching diagram.

Benchmarking

Benchmarking compares methods on the same data with the same metric, so we can see whether a small tree, a pruned tree, or an ensemble actually improves the result.

Why This Matters Here

The squirrel data is small enough to keep the story readable, but rich enough to show all four ideas before we move to larger urban-tree and wildfire examples.

Classification = choose a class label.
Regression = predict a numeric outcome.
Rules = express the tree as if/then statements.
Benchmarking = compare models on the same target and metric.
II. Rule-Based Trees

Rules Based Models

Rule-based models learn patterns from data, convert those patterns into logical rules, and then apply the rules to classify or predict outcomes.

How They Work
  • Learn patterns from data
  • Convert patterns into logical rules
  • Apply rules to classify or predict outcomes
Advantages
  • Easy to interpret
  • Transparent decision making
  • Often derived from decision trees
Disadvantages
  • Can create too many rules
  • Overfit data
  • Less accurate than more complex models

IV. Random Forests

Random Forests

Random forests keep the tree idea, but make it more stable by growing many trees and combining their predictions instead of trusting a single fit.

Why They Help

They reduce the correlation among trees, which makes the ensemble less noisy than bagging alone.

What Changes

Each split only sees a random subset of predictors, so the trees diversify and the vote becomes stronger.

When They Shine

They work especially well when there are many predictors and several of them are highly correlated.

IV. Random Forest Intuition

Random Forest

The core idea is still bagging, but each tree only sees a random subset of predictors at each split, which lowers correlation and improves the ensemble.

  • Ensemble Method: Built from many decision trees instead of one.
  • Randomness: Feature subsampling makes the trees different enough to help the vote.
  • Prediction Rule: Classification uses majority vote; regression uses averaged prediction.
  • Splits: At each split, RF considers a random subset of predictors
IV. Random Forest Intuition

Single Trees Are Easy To Read, But Often Too Unstable

A single decision tree can swing a lot when the training data changes a little. Random forests keep the interpretability of tree-based logic while reducing that variance through aggregation.

Single Tree

Easy to interpret, but small data changes can produce a very different structure and a different prediction path.

One tree can also overfit, because it has no built-in way to smooth over unstable split decisions.



Random Forest

Reduces correlation among trees to improve bagging predictive values and makes the final prediction more robust.

It works better as the number of predictors grows and the trees have more room to disagree in useful ways.

IV. Random Forest Mechanics

How Random Forest Works

Core Steps
  1. Start with the original training data.
  2. Create many bootstrap samples of the same size.
  3. Grow one tree per sample using a random subset of predictors at each split.
  4. Combine predictions by majority vote or averaging.
Forest View

The point is not to make one perfect tree. The point is to let many imperfect trees vote until their average is more stable than any one of them.

III. Bagging And Random Forests

Bagging And Random Forests Reduce Instability By Averaging Many Trees

The key tradeoff is that one tree is readable but unstable, so we average many trees and decorrelate them to reduce variance.

Bagging
ŷ(x) = (1 / B) Σ Tb(x)

Bootstrap the training data, fit one tree per resample, and average the predictions.

Random Forest
m ≈ √p

At each split, only a random subset of features is considered, which lowers correlation among trees.

Why It Helps
Lower variance

Small changes in training data can make one tree swing wildly. Averaging many trees makes predictions more robust.

Bootstrap resamples Out-of-bag checks Random feature subsets Majority vote or average

III. Cubist

Cubist Method Overview

Cubist is based on the M5 method created by Ross Quinlan. It partitions data with rules, fits linear models inside each region, and combines the results through committees.

Based On

The method builds on M5, which starts with rule-based partitions and then uses local linear models to sharpen the prediction.

Core Steps
  • Partition data based on specific rules
  • Fit a linear model for each node
  • Create committees that correct residuals from earlier models
  • Average predictions from all committees
  • Use k-nearest neighbors on the final prediction for adjustments
Why It Matters

Cubist keeps the structure of a tree, but adds local linear corrections and committee averaging to improve regression accuracy.

IV. Boosting

Adaptive Boosting

Adaptive boosting builds classifiers step by step, reweighting the misclassified samples so later learners focus on the hardest cases.

Binary Classification

The response is coded as a binary class, and a classifier G(x) predicts one of the two labels from the predictor matrix X.

How It Builds
  • Build classifiers Gm(x) one step at a time
  • Choose the best classifier at each iteration using current weights
  • Increase weights for misclassified samples
  • Lower weights for correctly classified samples
Final Prediction

Each classifier gets a stage weight from its error, and the final answer is a weighted combination of all Gm(x) values.

IV. Boosting Mechanics

Boosting Builds An Additive Model By Correcting Residuals In Sequence

The stagewise view matters here: each tree is a small correction to what the earlier trees still miss, so the animation mirrors the additive math.

Stagewise Additive Form
FM(x) = Σ ν fm(x)

Each new learner contributes a small correction rather than a completely independent prediction.

Interpretation

Bagging averages many independent trees. Boosting fits them one after another so each tree focuses on what remains difficult.

III. Benchmark Tradeoffs

Higher Accuracy Usually Costs More Time And More Tuning

This is where pruning and model complexity meet the benchmark chart: every performance gain has a cost in computation, tuning, or interpretability.

Cost-Complexity Intuition

Pruning limits size, depth, and unnecessary branches. In practice, wrappers like caret add cross-validation and tuning overhead on top of that search process.

  • Single trees are extremely fast.
  • Tuned wrappers are slower because they fit many candidate models.
  • Stronger ensembles tend to win on harder targets.
`caret` = Classification And REgression Training, which adds tuning and cross-validation.
Benchmarking = compare fit time, accuracy, and complexity on the same task.
