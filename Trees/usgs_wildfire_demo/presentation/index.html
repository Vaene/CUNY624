I. Introduction

Why Tree Models Work So Well For Mixed Environmental Data

The squirrel dataset is the perfect on-ramp because it mixes behavior, place, and simple categories without demanding heavy preprocessing.

Advantages From The PDF

  • They do not assume a fixed linear relationship between predictors and response.
  • They handle categorical and numeric predictors in the same workflow.
  • They stay usable with skewed, sparse, or partially missing data.
  • They are easy to explain because each prediction is a path through rules.

I. Split Mechanics

Classification Trees Choose Splits That Reduce Impurity Fastest

The PDF’s Gini example gives the math behind the first squirrel split: we want child nodes that are more homogeneous than the parent.

Gini
Gini = 1 - Σ pk2

For a two-class problem, low Gini means the node is mostly one class. Trees search for the split that gives the biggest drop in impurity.

Entropy
Entropy = - Σ pk log pk

Entropy tells the same story with a different scale: more mixed classes create more uncertainty, so good splits reduce entropy.

Worked Example
1/2 - 1/6 = 1/3

In the PDF student example, splitting on gender drops weighted Gini from 0.5 to 0.1667, which is a big impurity reduction. That is the same logic behind a split like Feeding in the squirrel tree.

How To Read A Tree

Each observation starts at the root, follows one branch at every rule, and ends in one leaf. One path equals one prediction.

Why This Matters Here

Squirrels make the logic visible because “ground or above ground” is intuitive enough that we can connect the formula to the branches live.

II. Regression Trees

Regression Trees Replace Impurity With RSS And Predict Regional Means

The PDF’s regression section fills in the math under the squirrel perch-height model and the wildfire acreage baseline.

Split Objective
RSS = Σ (yi - ŷRj)2

Regression trees partition the predictor space into regions Rj and predict the mean response inside each region.

Metrics Primer
MAE and RMSE

MAE keeps the original unit, while RMSE punishes larger misses more heavily. Lower is better for both.

III. Ensemble Mechanics

Bagging And Random Forests Reduce Instability By Averaging Many Trees

The PDF gives the ensemble intuition directly: one tree is readable but unstable, so we average many trees and decorrelate them.

Bagging
ŷ(x) = (1 / B) Σ Tb(x)

Bootstrap the training data, fit one tree per resample, and average the predictions.

Random Forest
m ≈ √p

At each split, only a random subset of features is considered, which lowers correlation among trees.

Why It Helps
Lower variance

Small changes in training data can make one tree swing wildly. Averaging many trees makes the predictions more robust.

Bootstrap resamples Out-of-bag checks Random feature subsets Majority vote or average

III. Bagging & Random Forests

Ensembles Help Most When The Urban Tree Task Gets Harder

For simple site-type prediction, one tree is already strong. The harder multi-class task of predicting TreeType is where ensembles begin to earn their keep.

Method Context

  • bagging fits many trees on bootstrap samples and averages them.
  • randomForest adds random feature selection to reduce tree-to-tree similarity.
  • Out-of-bag samples can estimate predictive performance without a separate validation pass.

III. Benchmark Tradeoffs

Higher Accuracy Usually Costs More Time And More Tuning

This is where the PDF’s pruning and complexity ideas meet the benchmark chart: every performance gain has a cost in computation or simplicity.

Cost-Complexity Intuition

Pruning limits size, depth, and unnecessary branches. In practice, wrappers like caret add cross-validation and tuning overhead on top of that search.

  • Single trees are extremely fast.
  • Tuned wrappers are slower because they fit many candidate models.
  • Stronger ensembles tend to win on harder targets.

Fit a linear model inside each region instead of using only a flat leaf average.

Average committees, then optionally adjust with nearest neighbors.

IV. Boosting Mechanics

Boosting Builds An Additive Model By Correcting Residuals In Sequence

The PDF’s stagewise view fits perfectly here: each tree is a small correction to what the earlier trees still miss.

Stagewise Additive Form
FM(x) = Σ ν fm(x)

Each new learner contributes a small correction rather than a whole new independent prediction.

Interpretation

Bagging averages many independent trees. Boosting fits them one after another so each tree focuses on what remains hard.

IV. Boosting

Gradient Boosting Wins On Error, But The Gain Comes With Extra Cost

This benchmark compares a single tree, a bagging-style ensemble, classic boosting, and a modern histogram-based variant on the same wildfire task.

Pros

Cons

Slower than one tree and less interpretable once many corrections are stacked.
